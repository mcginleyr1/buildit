# BuildIt Frontend Design

A modern, dark-first UI for CI/CD pipelines and multi-target deployments.

Inspired by: **Buildkite** (DAG visualization, build UI) and **Spacelift** (sidebar nav, deployment management)

---

## Design Principles

1. **Dark-first** - Optimized for dark mode, light mode as secondary
2. **Information density** - Show relevant data without overwhelming
3. **Real-time** - Live updates via WebSocket, no manual refresh needed
4. **Visual pipelines** - DAG visualization is a first-class feature
5. **Fast navigation** - Sidebar nav, keyboard shortcuts, command palette

---

## Tech Stack

- **Templating**: Askama (Jinja2-like, compile-time checked)
- **CSS**: Tailwind CSS (via CDN for now, build step later)
- **Interactivity**: htmx + htmx WebSocket extension
- **Icons**: Heroicons or Lucide
- **Fonts**: Inter (UI), JetBrains Mono (code/logs)
- **DAG Rendering**: SVG-based (D3.js or hand-rolled)

## DAG Visualization

### Requirements

1. **Render stages as nodes** - name, status, duration
2. **Render dependencies as edges** - directed arrows
3. **Layout algorithm** - automatic positioning (dagre-style)
4. **Interaction** - pan, zoom, click to select stage

### Implementation

Start with **SVG-based** rendering for simplicity:
- Easier to style with CSS
- Better accessibility
- Can migrate to Canvas later if needed
